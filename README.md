*** 所有程式碼透過 Google AI Studio Gemini 3 生成 ***

**Web App 前端與使用者介面 (UI/UX)**
 - HTML5 & CSS3
 - Responsive Design (響應式設計) 確保手機、平板與電腦上體驗一致。
 - 使用 Leaflet 底圖，匯入建築物邊界、車格、柱號 GIS 資訊。
 - 使用 SVG 套圖呈現文字、外觀及圖例。若須修改文字，直接於 Illustrator 修改後重新匯出即可。
 - 使用按鈕輸入，減少手機原生鍵盤輸入占用畫面的次數。

**Web App 功能摘要**
 - 導航演算法 (Pathfinding) 採用 Dijkstra 演算法，計算最短路徑。
 - 定義車道 GIS 資訊，於車格中點與道路投影距離後渲染路徑。
 - 靜態網站，僅在瀏覽器運作，不依賴外部資料庫，使用PWA，確保在地下停車場無訊號時，仍能運作。
 - 具羅盤功能，點擊右上角羅盤按鈕使用裝置陀螺儀資訊，在沒有 GPS 情況下進行定向。
 - 可於按鈕搜尋車格後，設為終點，再到畫面上選取目前所在的位置，使用彈出視窗設定為起點。

**解決問題**
 - 既有系統未提供柱號，亦無格號搜尋，所有檢索需先有車牌號碼，才能進行檢索。</br>
 --> 本系統無須輸入車牌號碼，即可對已知資訊進行搜尋定位。
 - 牆體及建築物結構過度強調，但間接遮蔽了方向性，產生隧道效應。</br>
 --> 僅對使用者肉眼可見的牆面進行標示，並著重於車道與牆體的關係，及車格號、柱號的相互位置對照。
 - 既有系統僅提供路徑，但無法對路徑進行拖拉、放大，或旁鄰車格或路徑上車格號碼交互檢視。</br>
 --> 可提供縮放，所有車格號碼可互相對應。雖方向恆向北邊，但可以啟動羅盤檢視現在行進間路徑是否方向一致。
